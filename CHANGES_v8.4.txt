AIMxASSIST v8.4 — Overlay Visual & AutoPlay Fixes
====================================================

BUGS FIXED IN v8.4
------------------

1. Ghost coin circles cluttered the board  [AimOverlayView.java]

   ROOT CAUSE: onDraw() rendered all CV-detected coins as semi-transparent
   white/black circles overlaid on the real board, creating confusing ghost
   discs on every frame.

   FIX: Removed the coin-drawing loop entirely. Only aim arrows and pocket
   markers are drawn. CV detection still uses coins internally for shot
   computation — they are just no longer rendered.

2. Aim lines cluttered the striker position  [AimOverlayView.java]

   ROOT CAUSE: Lines were drawn FROM the striker centre to the ghost-ball
   contact point, causing a confusing ray-burst pattern centred on the
   striker. The user could not see where to aim at a glance.

   FIX: Removed all striker→ghost line segments. Instead a DIRECTIONAL
   ARROW is drawn AT the ghost-ball endpoint (the "last point" of the shot):
     • Arrow tip points in the direction of striker travel
     • Short tail-stem shows which direction the striker came from
     • Soft glow ring highlights the endpoint
   Best shot: bright GOLD arrow. Ranks 2-5: progressively dimmer cyan/white.

3. Coin→pocket line is kept [AimOverlayView.java]

   The trajectory line showing where the targeted coin will travel to the
   pocket is preserved — it is the most useful visual cue for the player.

4. AutoPlay gesture unreliable  [AutoShootService.java]

   ROOT CAUSE: Swipe started exactly at the striker centre. Carrom Disc Pool
   treats this as a tap, not a shot gesture, and ignores it.

   FIX: Gesture now starts BEHIND the striker (25–55 px opposite to shot
   direction) and flicks THROUGH the striker toward the target. This matches
   the natural "flick" motion the game expects.

   Additional changes:
     • Swipe distance: 160 px (soft) → 460 px (full power). Old range
       120–360 px was too short on high-density screens.
     • Duration: 80–140 ms (was 60–120 ms). Avoids game input debounce.
     • Backstep scales with power (25–55 px) for proportional feel.
