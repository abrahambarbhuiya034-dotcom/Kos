AIMxASSIST v6.0 — Pure-Java Detector + Full Fix
================================================

ROOT CAUSE (why nothing worked in v5):
  BoardDetector depended on OpenCV (native C++ library).
  On most phones, OpenCV failed to load → cvReady stayed false →
  detect() always returned syntheticState() — a hardcoded frozen demo
  with coins at the same positions EVERY frame, regardless of where
  your actual striker or board was. That's why lines looked like a
  stuck photo and never followed your striker.

FIXES IN THIS VERSION:
──────────────────────

1. BoardDetector.java — COMPLETE REWRITE (pure Java, zero OpenCV)
   • Uses only Android's Bitmap.getPixels() — works on EVERY phone
   • Board detection: scans for orange/brown border pixels every 4 px
     → finds board bounding box dynamically from REAL screen pixels
   • Coin detection: scans inside the board every 3 px, classifies:
       white pixels  → white coins or striker
       black pixels  → black coins
       red pixels    → queen (red piece)
   • Greedy blob clustering groups nearby same-colour hits into coins
   • Striker = largest white blob in bottom 38% of board height
   • EMA smoothing prevents jitter between frames
   • APK is ~30 MB smaller (OpenCV removed)

2. MainApplication.java — OpenCV loading removed entirely
   • cvReady = true (constant) — no runtime loading, no failures

3. build.gradle — OpenCV dependency removed
   • No more JitPack / opencv AAR — cleaner, faster builds
   • APK is ~30 MB smaller

4. Floating icon — new crosshair/target design
   • Cyan glowing circle with crosshair lines
   • "AIM" text label at bottom so it's unmistakable
   • More visible against both dark and light backgrounds

HOW AIM LINES WORK NOW:
────────────────────────
Every 33 ms (30 fps):
  1. Screen captured → downscaled to 720 px wide
  2. BoardDetector scans for orange border → finds YOUR board
  3. Scans inside board for coins (white/black/red blobs)
  4. Identifies striker (big white circle in bottom zone)
  5. GameState flows into AimOverlayView
  6. AimOverlayView draws lines based on REAL detected positions
  7. Lines follow the striker as you slide it left/right ✓

AUTOPLAY REQUIREMENTS (unchanged, but now actually works):
───────────────────────────────────────────────────────────
1. Overlay ON (floating icon visible)
2. Auto-Detect ON (tap toggle → grant one-time screen capture)
3. Android Settings → Accessibility → AIMxASSIST → Enable
4. AutoPlay toggle ON
5. Hold board still for ~0.5 s → app auto-fires
